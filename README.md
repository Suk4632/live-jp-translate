# 🎙 일본어 라이브 방송 실시간 번역기

유튜브 라이브 등 **컴퓨터에서 나오는 일본어 소리를 실시간으로 한국어 자막**으로 보여주는 프로그램입니다.

```
유튜브 라이브 소리 → AI 음성 인식(Whisper) → 구글 번역 → 화면 위 자막
```

- **말하는 도중에도 약 1초마다 자막이 실시간으로 갱신**됩니다 (말하는 중엔 `⋯` 표시, 문장이 끝나면 확정 자막으로 교체)
- 방송 화면 위에 **항상 떠 있는 반투명 자막창**으로 표시됩니다 (마우스로 위치 이동 가능)
- API 키, 회원가입 전부 **필요 없음** (전부 무료)
- 음성 인식은 내 컴퓨터에서 돌아가고, 번역만 인터넷(구글 번역)을 사용합니다

---

## 1. 설치 (Windows 기준)

### ① 파이썬 설치
1. https://www.python.org/downloads/ 에서 최신 파이썬을 내려받아 설치
2. 설치 첫 화면에서 **"Add Python to PATH" 체크박스를 꼭 켜세요!** (제일 중요)

### ② 번역기 설치
이 폴더 안의 **`설치.bat` 을 더블클릭**하면 끝. (몇 분 걸립니다)

> 검은 창에 "설치 완료!"가 나오면 성공입니다.

---

## 2. 사용법

1. 유튜브에서 좋아하는 유튜버의 라이브 방송을 켭니다 (소리가 스피커로 나오게)
2. **`실행.bat` 을 더블클릭**합니다
3. 처음 실행하면 AI 모델을 자동으로 내려받습니다 (약 500MB, 한 번만)
4. "방송 소리를 기다리는 중..." 자막창이 뜨고, 유튜버가 말하면 몇 초 뒤에 한국어 자막이 나옵니다!

- 자막창은 **마우스로 끌어서** 원하는 위치로 옮길 수 있어요
- 종료: 자막창에서 **ESC** 키 또는 오른쪽 위 **✕**

> 💡 이어폰/헤드셋을 써도 됩니다. "스피커로 재생되는 소리"를 컴퓨터 내부에서 바로 캡처하기 때문에 마이크로 소리를 들려줄 필요가 없어요.

---

## 3. 자주 쓰는 옵션

`실행.bat` 대신 명령 프롬프트(cmd)에서 이렇게 실행할 수 있습니다:

| 명령 | 설명 |
|---|---|
| `python main.py` | 기본 실행 |
| `python main.py --model tiny` | **컴퓨터가 느려서 자막이 밀릴 때** (인식 품질은 조금 낮아짐) |
| `python main.py --model medium` | 컴퓨터가 좋을 때, 더 정확한 인식 |
| `python main.py --font-scale 1.5` | 자막 글자 1.5배 크게 |
| `python main.py --no-japanese` | 일본어 원문 없이 한국어만 표시 |
| `python main.py --no-overlay` | 자막창 없이 검은 콘솔 창에만 출력 |
| `python main.py --log 방송기록.txt` | 자막을 파일로 저장 |
| `python main.py --partial-interval 0` | 말하는 중 자막 갱신 끄기 (문장 끝나고만 표시) |
| `python main.py --list-devices` | 오디오 장치 목록 확인 |
| `python main.py --device "이름"` | 특정 오디오 장치 지정 |

> 💡 `python` 명령이 안 먹히면 `py` 로 바꿔서 실행하세요 (예: `py main.py --model tiny`).

모델 크기 가이드 (`--model`):

| 모델 | 속도 | 정확도 | 추천 |
|---|---|---|---|
| tiny | 매우 빠름 | 낮음 | 오래된 노트북 |
| base | 빠름 | 보통 | 사양이 낮은 컴퓨터 |
| **small** | 보통 | 좋음 | **기본값, 대부분 추천** |
| medium | 느림 | 매우 좋음 | 고사양 PC / 그래픽카드 |
| large-v3 | 매우 느림 | 최고 | NVIDIA 그래픽카드 필수 |

> 💡 NVIDIA 그래픽카드가 있으면 자동으로 사용해서 훨씬 빨라집니다.

## 4. 문제 해결

### 🔇 자막이 안 나와요 / 소리를 못 잡아요
1. 방송 소리가 실제로 스피커로 나오고 있는지 확인 (음소거 해제, 볼륨 올리기)
2. `python main.py --list-devices` 로 장치 목록 확인
3. 소리가 나오는 장치가 기본 스피커가 아니면: `python main.py --device "장치이름일부"`
4. 소리가 작으면 감지 민감도를 낮춰보세요: `python main.py --vad-threshold 0.005`

### 🐢 자막이 너무 늦게 나와요 / 실시간 갱신이 안 돼요
- 말하는 중 자막(`⋯` 표시)은 **컴퓨터가 인식하는 속도만큼** 갱신됩니다. 갱신이 뜸하면 컴퓨터가 인식을 따라가지 못하는 것이니 `python main.py --model base` 또는 `--model tiny` 로 실행해 보세요 (훨씬 빨라짐)
- NVIDIA 그래픽카드가 있는 컴퓨터라면 자동으로 사용되어 매우 빨라집니다
- 확정 자막(밝은 흰색)은 문장이 끝난 것을 감지한 뒤에 나오므로 약간의 시차가 정상입니다

### 🌐 "번역 실패"라고 나와요
- 번역은 구글 번역(인터넷)을 사용합니다. 인터넷 연결을 확인하세요.

### 🍎 macOS 사용자
- 맥은 시스템 소리 캡처를 기본 지원하지 않아 [BlackHole](https://github.com/ExistentialAudio/BlackHole) 같은 가상 오디오 장치를 설치한 뒤 `--device "BlackHole"` 로 지정해야 합니다.

### 🐧 Linux 사용자
- PulseAudio/PipeWire의 "Monitor" 장치를 자동으로 찾습니다. 안 되면 `--list-devices` 로 확인 후 `--device "monitor"` 지정.

### 이상한 문장이 가끔 나와요 ("ご視聴ありがとうございました" 등)
- AI가 무음/음악 구간에서 가끔 문장을 지어내는 현상(환각)입니다. 흔한 것들은 자동으로 걸러내지만 완벽하지는 않아요. 노래 구간에서 특히 자주 생깁니다.

## 5. 파일 구성

```
jp-live-translator/
├── 설치.bat            ← 처음 한 번 더블클릭 (라이브러리 설치)
├── 실행.bat            ← 번역기 시작
├── main.py             ← 실행 진입점 (옵션 처리)
├── requirements.txt    ← 필요한 라이브러리 목록
└── translator/
    ├── audio.py        ← 스피커 소리 캡처 + 문장 단위로 자르기
    ├── asr.py          ← Whisper 일본어 음성 인식
    ├── translate.py    ← 구글 번역 (일→한)
    ├── overlay.py      ← 화면 위 자막창
    └── app.py          ← 전체 조립
```
